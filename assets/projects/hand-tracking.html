Overview

Hand Tracking Streamer (HTS) turns a Meta Quest headset into a practical hand-telemetry device for robotics. It streams 21-landmark hand tracking plus a 6-DoF wrist pose to your workstation over Wi-Fi.

Use UDP for low-latency teleoperation and TCP for reliable logging and dataset capture. HTS is free on the Meta Quest Store (and SideQuest), with Python + ROS SDKs for fast integration.

Why I Built It

HTS started as a need for reliable hand telemetry for teleoperating my dexterous hand (BiDexHand) and later became part of my LeVR pipeline.

I kept seeing the same pattern: researchers and developers reinventing fragile VR-to-robot glue code - often gated behind “developer mode” workflows. HTS packages the boring infrastructure (config, streaming, debugging) into a repeatable tool anyone can install and use with one click.

SDK & Extensions

HTS ships with a Python SDK (PyPI) and a ROS 2 bridge package, so streamed packets become usable data quickly—typed parsing, logging, and visualization included.

The goal is to support teleop, imitation learning, and simulation workflows without bespoke plumbing. Docs and demos are included, and contributions are welcome (issues/PRs for integration, formats, or tooling).
